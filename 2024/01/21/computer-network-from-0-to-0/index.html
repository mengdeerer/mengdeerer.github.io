计算机网络从入门到放弃

概述

tcp/ip 五层原理结构

  • 应用层: http
  • 传输层: tcp udp
  • 网络层: ip
  • 数据链路层: 光纤
  • 物理层：wifi

    服务垂直，协议水平

物理层

传输媒体

  • 导向型
    • 同轴电缆，双绞线，光纤
    • 光纤：电信号转换成光信号在光纤中传输
  • 非导向型（自由空间）
    • 无线电波，微波，可见光等

传输方式

  • 串并行
    • 串行：只有一条数据线传输
    • 并行：多线传输，短距离
  • 同步异步传输
    • 同步传输：时钟同步，以比特流方式传输，中间无间隔
    • 异步传输：字节传输，两相邻字节间有间隔，并且每个字节间有起始位和终止位
  • 通信
    • 单向通信：无双方交互
    • 双向交替通信：发送接收不可同时进行
    • 双向同时通信：同时发送接收消息

编码和调制

消息->数据->信号->数字基带信号->基带调制（数字信道），带通调制（模拟信道）
码元：信号编码单元，即在使用时间域的波形表示信号时，代表不同离散数值的基本波形称为码元

编码方式

  • 不归零制
    • 正电平表示 1，负电平表示 0，效率高但是存在通信双方同步问题
  • 归零制
    • 信号在每个码元期间会回归到零电平，带宽在传输归零中浪费了
  • 曼彻斯特编码
    • 每个码元中间时刻会发生跳变，电平的跳变可以进行时钟同步，向上还是向下跳变分别表示 0，1(自己规定)
  • 差分曼彻斯特编码
    • 中间时刻发生跳变，并且仅表示时钟同步，数据的表示在每个码元一开始处是否有电平跳变，有跳变表示 1，无跳变表示 0

调制

  • 基带调制：数字信号波形变换，仍是数字信号，也称为编码
  • 带通调制：将数字信号的频率范围搬移到较高的频段，并转换为模拟信号

带通调制方法

基本：调幅，调频，调相
混合：一个码元可以表示多个 bite 信息量
使用正交振幅调制，例 QAM-16，根据不同相位和振幅可以调制出 16 种不同的码元，每个码元可以表示个 bite 信息量，相邻码元使用格雷码进行编码，只有一位不相同

信道复用技术

频分复用

不同用户占用不同频带资源发送数据

时分复用

用户不同时间占用同样的信道资源发送数据

统计时分复用：数据发送给集中器，集中器对输入数据按照顺序进行缓存，如果一个 STDM 帧数据满了就发送出去

码分复用
